实现一个请求

注册一个路由（urls.py）

  • urlpatterns参数

    • 参数①：匹配规则 正则

    • 参数②：视图函数 对应的是views.py中的一个函数，不加函数括号

      urlpatterns = [
    path('admin/', admin.site.urls),
    # 设置了一个hello路径，然后传给App中的view，调用view中hello函数（不要括号）
    path('hello/', views.hello),
]

实现对应的视图函数（views.py）

  • 创建的函数第一个参数是request

  • 永远记得返回Response或者他的子类

    def hello(request):
    return HttpResponse("刘辟")

实现访问网页

第一种方式：模板放在创建的应用APP中

  1. 在应用中创建一个文件，一般名为templates，里面放网页
  2. 标记为模板文件夹

  3. 在设置（setting.py）中 INSTALLED_APPS 添加该应用（不然不知道有这个应用，还有添加模板的路径

    INSTALLED_APPS = [
        'django.contrib.admin',
        'django.contrib.auth',
        'django.contrib.contenttypes',
        'django.contrib.sessions',
        'django.contrib.messages',
        'django.contrib.staticfiles',
        'App'  //添加App的名称
]    

TEMPLATES = [
{
    'BACKEND': 'django.template.backends.django.DjangoTemplates',
    # 路径
    'DIRS': [
        # BASE_DIR是该项目的路径，'App/templates'是你模板的路径
        os.path.join(BASE_DIR, 'App/templates'),
    ],

    .....

}
  1. 在路径（urls.py）中添加视图(views.py)函数

  2. 视图函数用返回 render（request，”网页全名”） 注：render可以帮我们把网页渲染

    def index(request):
    return render(request,'index.html')

第二种方式：模板放在创建的APP外

  1. 与第一种步骤的 1 ， 2 ， 4 ， 5 一样

  2. 在路径（urls.py）TEMPLATES 里面的 DIRS 添加该模板的路径

    # 模板
TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        # 路径
        'DIRS': [
            # 把模板路径添加进来，这里灵活获取该路径，当然可以是绝对路径放里面（不建议）
            os.path.join(BASE_DIR , 'templates'),
            'D:\Python\PyCharm\HelloDjango\templates'， # 不建议这样写
        ],
        'APP_DIRS': True,
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',

            ],
        },
    },
]

拆分项目的路由（urls.py）

目的

  • 不然以后的所有应用的url都在主项目里面，不便于写看和写

拆分

  • 原理：
    • 让每个APP都有自己的view和url
    • 再通过主项目的url来联系创建应用的url

  • 步骤

    • ①：创建的应用添加路由（urls.py）和视图（views.py） 注：和主项目的一样，包要正确哦
    • ②：应用的路由调用视图

    • ③：主项目的路由调用应用的路由

      path('two/', include('应用名.urls')) # 这里是urls，提示是urls.py,点击实际是urls，没的错，不用添加 .py ,添加了反而是错的

    • ④：网页通过 根路由/子路由 来获取

      例子：
http://127.0.0.1:8000/two/kjj
two:是应用名
kjj：是应用中url中的路径匹配
