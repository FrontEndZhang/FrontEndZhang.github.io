Less基础

发布 : 2019-07-16 分类 : Less 浏览 :

1.为什么要用less，less是什么：

  • 我的体会是：由于一个项目，css代码很多，一百多行都能让你看的眼花缭乱，还要去慢慢找，很麻烦，而且结构不清晰。所以，为我们就选择less来增强css的结构性和可阅读性，我们只需要看less文件，就不需要在看css文件。

  • Less 是一个Css 预编译器,意思指的是它可以扩展Css语言,添加功能如允许变量(variables),混合(mixins),函数(functions) 和许多其他的技术，让你的Css更具维护性，主题性，扩展性。

2.less的安装环境

  1. less 既可以在客户端上运行，也可以借助Node.js在服务器上运行

  2. 介绍一个简单的方法：

    • 下载安装好node.js
    • 用vscode下载插件Easy Less
    • 重启vscode，然后编写less的文件，vscode就会给你自动在该文件夹里面生成对应的css文件

  3. 其他安装在网上查看即可 http://lesscss.cn/#using-less

3.less的注释

  1. 以//开头的注释，不会编译到css文件中
  2. 以/**/包裹的注释会编译到css文件中

4.less中的变量

  • 使用@来声明一个变量：@color:pink; @pro:margin;
    1. 作为普通属性值来使用：直接使用@pink
    2. 作为选择器和属性名： @{selector的值}的形式，比如 @{pro}: 0;
    3. 作为URL：@{url}
    4. 切记：变量的延迟加载

5.less中的嵌套规则

  1. 基本嵌套规则

    • html代码：

      <div class="container">
    <div class="inner"></div>
</div>

    • less文件源码

      .container {
    height: 100px;
    width: 100px;
    .inner {
        height: 50px;
        width: 50px;
        }
}

    • 输出的css源码：

      .container {
  height: 100px;
  width: 100px;
}
.container .inner {
  height: 50px;
  width: 50px;
}

  2. &的使用

    • 个人认为，该符号类似于Java中的this，代表自己

    • 使用方法：

      .container {

      height: 50px;
width: 50px;
  &:hover {  //让自己获得hover效果
          background: yellow;
  }

      }

6.less中的混合

混合就是将一系列属性从一个规则集引入到另一个规则集的方式

1.普通混合（没有带括号）

//该段代码会被编译到css文件中
.common {
        position: @p;
        height: @h;
        width: @w;
}
.container {
    .common //输出的css中也包含common中的代码

}

2.不带输出的混合（后面有括号）

.common() {  //该段代码不会被编译到css文件中
         position: 100px;
         height: 100px;
         width: 100px;
}


.container {
    .common //输出的css中也包含common中的代码

}

3. 带参数的混合

.common(@h,@w,@p) {
             position: @p;
             height: @h;
             width: @w;
}
.container {
    .common(50px,50px,ralative) //输出的css中也包含common中的代码

}

4. 带参数并且有默认值的混合

.common(@h:100px,@w:100px,@p:absolute) {
             position: @p;
             height: @h;
             width: @w;
}
.container {
    .common() //输出的css中也包含common中的代码,默认为100px,100px,absolute

}

5. 命名参数(指定给哪个参数传值)

.common(@h:100px,@w:100px,@p) {
             position: @p;
             height: @h;
             width: @w;
}
.container {
    .common(@p:absolute) 

}

6. 匹配模式

.common(signA,@h:100px,@w:100px,@p:absolute) { 
             position: @p;
             height: @h;
             width: @w;
}
.common(signB,@h:200px,@w:200px,@p:relative) { 
             position: @p;
             height: @h;
             width: @w;
}
.container {
    .common(signB) //选择的是signB中的代码片段

}

7. arguments变量

//less代码
.common(@a,@b,@c) {
       border: @arguments  
}
.container {
    .common(1px,solid,red); //等同于 
}
//输出的css代码如下：
.container {
  border: 1px solid red;
}

7.less运算

  • 在less中可以进行加减乘除的运算，但是至少要加一个单位哦！

8.less避免编译

  • 语法： ~”直接显示的内容”

9.less继承 (继承让相同的代码不再重复出现css里面)

//less文件
    .common{
        height: 100%;
        width: 50%;
        &:hover{  //注意
            background: yellow;
        }
    }
    .container {
        .right:extend(.common all) { //all代表着有关的代码全部继承下来，比如：hover等等
            background: blue;
        }
        .left:extend(.common all) {  
            background: green;
        }
    }

//css输出
    .common,
    .container .right,
    .container .left {
      height: 100%;
      width: 50%;
    }
    .common:hover,
    .container .right:hover,
    .container .left:hover {
      background: yellow;
    }

    .container .right {
      background: blue;
    }
    .container .left {
      background: green;
    }
