Mybatis

发布 : 2019-10-10 浏览 :

mybatis环境搭建

1.创建maven工程并导入坐标

<dependency>
        <groupId>org.mybatis</groupId>
        <artifactId>mybatis</artifactId>
        <version>3.5.1</version>
</dependency>

2.创建实体类和dao的接口

//实体类继承Serializable
public class User implements Serializable {

private Integer id;
private String username;
private Date birthday;
private String sex;
private String address;

//dao层接口
public interface IUserDao {
        void findAll();
}

3.创建Mybatis的主配置文件SqlMapConfg.xml

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <!--  配置环境  -->
    <environments default="mysql">
        <!-- 配置mysql的环境,注意，default和id要一致 -->
        <environment id="mysql">
            <!--  配置事务类型  -->
            <transactionManager type="JDBC"/>
            <!--   配置数据源（连接池）  -->
            <dataSource type="POOLED">
                <!--  配置连接数据库的基本信息  -->
                <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql://localhost:3306/mybatis"/>
                <property name="username" value="root"/>
                <property name="password" value="117139"/>
            </dataSource>
        </environment>
    </environments>
    <!--  指定映射配置文件的位置，映射配置文件指的是每个dao独立的配置文件  -->
    <mappers>
        <mapper resource="xyz/zjy/dao/IUserDao.xml"/>
    </mappers>
</configuration>

4.创建映射配置文件IUserDao.xml

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<!-- namespce接口 -->
<mapper namespace="xyz.zjy.dao.IUserDao">
    <!--  查询所有，id为接口的方法，resultType：指定返回的封装成的实例  -->
    <select id="findAll" resultType="xyz.zjy.domin.User">
        select * from user
    </select>
</mapper>

5.创建实例并执行

public void myTest() throws IOException {
    //1.读取配置文件
    InputStream in = Resources.getResourceAsStream("SqlMapConfig.xml");
    //2.创建SqlSessionFactory工厂
    SqlSessionFactoryBuilder builder = new SqlSessionFactoryBuilder();
    SqlSessionFactory factory = builder.build(in);
    //3.使用工厂生产SqlSession对象
    SqlSession session = factory.openSession();
    //4.使用SqlSession创建Dao接口的代理对象
    IUserDao user = session.getMapper(IUserDao.class);
    //5.使用代理对象执行方法
    List<User> userAll = user.findAll();
    System.out.println(userAll);
    //6.释放资源
    session.close();
    in.close();
}

注解开发

主配置文件更改

 将xml配置中的<mapper resource>，改为<mapper class>,并指定dao接口的全限定类名
 <mappers>
    <mapper class="xyz.zjy.dao.IUserDao"></mapper>
</mappers>

添加注解

//在接口的方法上添加注解和Sql语句
@Select("select * from user")
List<User> findAll();

注意事项

1.名称

1.    创建IUserDao.xml和IUserDao.java时名称一致是为了好理解，类比理解
2.    在Mybatis中他把持久层的操作接口名称和映射文件也叫做：Mapper，所以，IUserDao和IUserMapper是一样的

2.目录

在idea中创建目录的时候，它和包是不一样的
包在创建时：xyz.zjy.dao是三级结构
目录在创建时：xyz.zjy.dao是一级目录

3.结构

mybatis的映射配置文件位置必须和dao接口的包结构相同

4.映射文件中namespace属性

映射配置文件的mapper标签namespace属性的值必须是dao层接口的权限定类名

5.映射文件中id属性

映射配置文件的操作配置（select，update这些标签的id属性），id属性的取值必须是dao接口的方法名

<select id="findAll">
    select * from user
</select>

6.可以无需dao的实现类

当我们遵从了注意事项的三四五点之后，在开发中就可以不写dao的实现类

出现的Bug

1.主配置文件中default和id不一致：

  • 问题描述：

    Error opening session. Cause: java.lang.NullPointerException

  • 解决方法：

    default和id要一致
<environments default="mysql">
    <environment id="mysql">

2.封装实例未指定

  • 问题描述：

    Error querying database.  Cause: org.apache.ibatis.executor.ExecutorException: A query was run and no Result Maps were found for the Mapped Statement 'xyz.zjy.dao.IUserDao.findAll'.  It's likely that neither a Result Type nor a Result Map was specified

  • 解决方法：

     添加resultType：返回的封装成什么实例
<select id="findAll" resultType="xyz.zjy.domin.User">

3.使用了xml开发之后再使用注解开发出现问题

  • 问题描述：

    //这个问题产生是因为xml开发后，mybatis会自动把映射配置xml文件放在和接口类同一个文件中（如图1）
Cause: org.apache.ibatis.builder.BuilderException: Error parsing SQL Mapper Configuration. Cause: java.lang.IllegalArgumentException: Mapped Statements collection already contains value for xyz.zjy.dao.IUserDao.findAll. please check xyz/zjy/dao/IUserDao.xml and xyz/zjy/dao/IUserDao.java (best guess)

  • 解决方法 

    这时候删除该编译之后的target项目中,和接口类同目录下的xml文件即可
